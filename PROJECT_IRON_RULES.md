# Video Create Studio 项目铁律

> 本文档定义项目的核心开发规则与防退化要求。所有后续功能、重构、修复、文档更新都必须遵守。

---

## 铁律一：Agent 必须亲自完成分析与实现

**规则：** 在协助开发时，必须直接阅读代码、理解逻辑、完成修改与修复。禁止编写自动化脚本去调用外部 AI 接口，再把结果回写到仓库里，替代本应由 Agent 自己完成的分析和实现工作。

**正确做法：**

- 直接阅读源码，定位问题，完成实现。
- 直接修改项目文件，并自行验证结果。

**禁止做法：**

- 写一个 Python 或 Node 脚本，把代码发给外部 AI 处理后再写回仓库。
- 把“分析代码逻辑”外包给另一个未受控的远程服务。

---

## 铁律二：跨进程通信必须极简

**规则：** Tauri 前端、Rust 后端与 Python 渲染引擎之间的通信，必须坚持“只传路径与状态，不传媒体数据”。

**强制要求：**

- 禁止通过 IPC 传输视频、图片、音频的二进制内容。
- 仅允许传递绝对路径、轻量参数和结构化状态信息。
- Python 进度反馈必须通过标准输出发送 JSON 行，例如 `{"progress": 50}`。
- 前端只消费状态和结果路径，不承担大媒体对象缓存职责。

---

## 铁律三：保持成品级 UI/UX 质量

**规则：** 前端实现不能退化成原生默认样式或临时拼接的视觉方案。

**强制要求：**

- 新界面必须沿用项目已有视觉系统与 CSS 变量。
- 所有关键交互都要有明确的反馈，例如 hover、loading、disabled、progress。
- 加载过程必须可感知，不能让用户在无状态反馈下等待。
- 新设计要优先保持统一、稳定、成品感，而不是只求“能用”。

---

## 铁律四：素材显示比例零拉伸、零变形

**规则：** 所有照片、视频素材在最终成片中的显示几何必须与用户预期一致。无论走 MoviePy、FFmpeg、缓存、预览、快速路径还是最终渲染，都不允许为了速度牺牲显示比例。

**强制要求：**

- 图片必须先做 EXIF 朝向修正，再按原始宽高比缩放。
- 视频必须尊重显示宽高、旋转信息、`sample_aspect_ratio`、`display_aspect_ratio` 等元数据。
- 缩放只能使用等比缩放，例如 `min(target_w / source_w, target_h / source_h)`，或 FFmpeg 的 `force_original_aspect_ratio=decrease`。
- 画布空白区域只能通过补边、模糊背景、纯色背景或设计背景解决。
- 任何 FFmpeg 快路径都必须显式设置 `setsar=1`。
- 性能优化只能降低动效复杂度或切换实现路径，不能改变素材几何关系。

**禁止做法：**

- 直接使用 `clip.resize((target_w, target_h))` 强行拉满画布。
- 仅按编码宽高处理视频，忽略显示比例元数据。
- 为了提速跳过显示比例检测与归一化步骤。

**回归验证：**

- 涉及渲染、转场、缓存、FFmpeg、MoviePy 的改动后，必须运行 `python tests\smoke_v5_video_geometry.py`。

---

## 铁律五：所有 Markdown 文档必须使用中文且禁止乱码

**规则：** 仓库内由项目维护的 `.md` 文档，默认以中文作为说明语言，并统一使用 UTF-8 编码保存，禁止出现乱码、错码、混乱编码或半翻译状态。

**强制要求：**

- 项目自有 Markdown 文档内容默认使用中文撰写。
- 保留英文仅限命令、路径、类名、函数名、字段名、协议名等技术标识符。
- 所有 Markdown 文件必须以 UTF-8 编码保存。
- 更新文档时要检查标题、正文、表格、代码块旁注是否出现乱码，例如常见错码片段 `鍙`、`乱码占位符`、重复问号占位等异常内容。
- 新增文档必须直接写成中文，不允许先写英文再长期搁置不翻译。

**优先处理范围：**

- 根目录文档
- `docs/`
- `archive/2026-05-cleanup/` 下保留的项目设计与归档说明
- `src-tauri/bin/README.md`

## 历史问题与防退化清单

在触碰相关模块前，必须先自检，避免旧问题复发。

### 1. Python 运行环境依赖缺失

- 风险点：Python 引擎依赖 `moviepy`、`ffmpeg`、`PIL` 等组件。
- 准则：修改引擎依赖后，必须同步更新 `README.md` 与相关安装说明，并补充清晰的依赖检查或失败提示。

### 2. FFmpeg 路径硬编码

- 风险点：历史上曾出现写死 `/usr/bin/ffmpeg`，导致 Windows 无法运行。
- 准则：必须通过 `shutil.which("ffmpeg")` 或打包资源路径进行动态定位。

### 3. IPC 消息丢失

- 风险点：Python 进程启动后过快输出，Rust 还未开始监听就丢失了前几行日志。
- 准则：Rust 在 `spawn` 子进程后应立即启动异步读取循环。

### 4. 文档编码混乱

- 风险点：历史文档中曾出现中文说明被错误编码后显示成乱码。
- 准则：所有文档改动提交前，必须确认内容为 UTF-8 中文可读文本。

| --- | --- | --- |
| `video_engine_v5.py` | V5 扫描、规划、编译、渲染核心引擎 | 修改时优先考虑稳定性、性能与缓存兼容性 |

| `src/styles.css` | 全局视觉基线 | 避免临时样式污染统一设计语言 |
| `docs/` | 当前项目设计与策略文档 | 必须保持中文、UTF-8、无乱码 |
