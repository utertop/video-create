# 错误码索引

## 目的

这份文档给 `Video Create Studio` 当前已接入的关键错误码一个统一索引，方便：

- 用户截图反馈时快速定位问题
- 支持同学按错误码归类排障
- 后续把错误码继续沉淀成更完整的支持文档体系

说明：

- `E_*` 表示错误
- `W_*` 表示警告或风险提示

---

## 一、项目与文档

### `E_PROJECT_DIR_MISSING`

- 含义：项目目录 `.video_create_project` 不存在或不可访问
- 常见场景：恢复最近项目时，输出目录被移动、删除或改名
- 建议处理：确认输出目录仍存在；如果目录已丢失，重新扫描素材并生成新项目

### `E_PROJECT_DOC_READ_FAILED`

- 含义：项目 JSON 文件存在，但读取失败
- 常见场景：权限不足、磁盘异常、路径不可访问
- 建议处理：确认目录可读；将项目移到本地可写路径后重试

### `E_PROJECT_DOC_INVALID_JSON`

- 含义：项目 JSON 文件已损坏或格式不合法
- 常见场景：手工编辑、异常中断、旧版工具输出不完整
- 建议处理：优先尝试恢复其他文档；如无法恢复，重新扫描并重新编译

### `E_PROJECT_DOC_TYPE_MISMATCH`

- 含义：项目 JSON 的 `document_type` 与文件名/预期用途不匹配
- 常见场景：文件被错误覆盖、手工拷贝错位
- 建议处理：检查 `media_library.json / story_blueprint.json / render_plan.json` 是否被错误替换

### `E_PROJECT_DOC_REWRITE_FAILED`

- 含义：旧版项目已识别，但自动迁移后的文档无法写回
- 常见场景：目录只读、磁盘权限受限、杀毒软件占用
- 建议处理：确认项目目录可写，再重新恢复项目

## 二、输入、输出与预检

### `E_OUTPUT_DIR_REQUIRED`

- 含义：缺少输出目录
- 常见场景：开始渲染前未选择输出目录
- 建议处理：先选择输出目录再继续

### `E_OUTPUT_NOT_WRITABLE`

- 含义：输出目录或输出文件所在目录不可写
- 常见场景：选择了只读目录、系统目录、受权限限制的路径
- 建议处理：改到桌面、文档或其他明确可写目录

### `E_MEDIA_SOURCE_MISSING`

- 含义：渲染计划引用的素材文件缺失
- 常见场景：编译后素材被移动、删除、重命名
- 建议处理：恢复素材原路径，或重新扫描素材并重新编译渲染计划

### `E_RENDER_PLAN_INVALID_JSON`

- 含义：`render_plan.json` 无法解析
- 常见场景：手工改坏、旧版本异常输出、文件损坏
- 建议处理：重新确认蓝图并生成新的渲染计划

### `E_PREFLIGHT_CHECK_FAILED`

- 含义：渲染前预检未通过
- 常见场景：输出目录不可写、素材缺失、渲染计划异常
- 建议处理：查看预检卡片中的具体子项错误码和提示

### `W_LARGE_RENDER_PLAN`

- 含义：检测到长视频或大量片段
- 常见场景：长时长、多素材项目
- 建议处理：优先使用 `stable` 档位，并保留导出的诊断包以便后续性能分析

## 三、Worker 与渲染执行

### `E_WORKER_ENTRYPOINT_MISSING`

- 含义：桌面端没有找到可用的 worker 启动入口
- 常见场景：打包不完整、worker 未生成、资源目录异常
- 建议处理：运行 `npm run check`，确认 worker 打包和桌面资源链路正常

### `E_WORKER_HEALTH_FAILED`

- 含义：worker 健康检查失败
- 常见场景：FFmpeg/Python 依赖异常、打包的 worker 无法正常启动
- 建议处理：运行 `npm run check`；必要时重新打包 worker

### `E_TASK_ALREADY_RUNNING`

- 含义：已有渲染任务正在运行
- 常见场景：重复点击开始渲染，或队列中已有活动任务
- 建议处理：等待当前任务结束，或先取消当前任务

### `E_TASK_CANCELLED`

- 含义：任务被用户取消
- 常见场景：主动停止渲染
- 建议处理：通常无需排障；如频繁取消，建议导出诊断包分析耗时阶段

### `E_RENDER_INTERNAL`

- 含义：渲染命令层内部异常
- 常见场景：Rust/Tauri 侧调用 worker 时出现未预期异常
- 建议处理：导出诊断包，并结合日志和最近一次操作复盘

### `E_SCAN_INTERNAL`

- 含义：扫描阶段内部异常
- 常见场景：素材目录访问异常、worker 协议异常
- 建议处理：确认目录可读，并附带诊断包反馈

### `E_PLAN_INTERNAL`

- 含义：蓝图生成阶段内部异常
- 常见场景：媒体库文档异常、worker 协议异常
- 建议处理：重新扫描素材；若持续复现，导出诊断包

### `E_COMPILE_INTERNAL`

- 含义：渲染计划编译阶段内部异常
- 常见场景：蓝图结构异常、旧文档字段缺失、worker 执行失败
- 建议处理：优先重新保存蓝图后再编译；若仍失败，导出诊断包

## 四、启动与环境

### `E_STARTUP_CHECK_FAILED`

- 含义：启动自检未通过
- 常见场景：资源文件缺失、worker 启动失败、缓存目录不可写
- 建议处理：查看启动自检卡片中的子项错误码，逐项修复

### `E_ENGINE_RESOURCE_MISSING`

- 含义：主渲染引擎脚本或资源不存在
- 常见场景：打包资源缺失、工作目录不完整
- 建议处理：检查安装包完整性或重新构建桌面端

### `E_DIRECTORY_NOT_WRITABLE`

- 含义：关键应用目录不可写
- 常见场景：缓存目录、应用数据目录、输出目录权限异常
- 建议处理：检查系统权限、杀毒软件限制或目录只读状态

---

## 五、支持建议

如果用户反馈问题，建议至少一起收集：

- 错误码
- 错误出现阶段：`scan / plan / compile / render / startup`
- 触发问题前最后一步操作
- `导出诊断包`

当前阶段最推荐的组合是：

1. 让用户截图错误卡或预检卡
2. 记录错误码
3. 让用户导出诊断包
4. 再决定是否需要远程复盘项目目录
