# 章节文字风格与动效预览设计方案

## 背景

当前蓝图审核页已经支持为每个章节选择文字风格和入场动效，但创作者只能看到下拉框名称，例如“旅行明信片”“淡入上移”。这些名称不足以判断最终画面效果。

对于视频制作来说，完整渲染后才发现风格不合适，会浪费大量时间，尤其是几千张图片、几十个视频的大项目。因此需要在进入最终渲染前，提供低成本、可视化、可试错的预览能力。

## 用户目标

用户需要在蓝图审核阶段完成三件事：

1. 快速知道当前章节文字风格和动效大概长什么样。
2. 能够点击查看更接近真实视频的动态效果。
3. 能够集中浏览全部风格和动效，自由组合后再回到章节里选择。

## 两个方案对比

### 方案 A：在每个章节卡片内显示效果缩略图

在“风格 / 动效”选择器附近显示一个小预览缩略图，点击后打开当前章节的动效预览。

优点：

- 离当前章节最近，用户不需要离开蓝图审核页。
- 每个章节可以直接看到当前选择的大致效果。
- 很适合快速检查“这个章节是不是看起来对味”。

缺点：

- 如果每个章节都显示动态预览，会增加页面复杂度。
- 大项目里章节很多，全部生成动态缩略图可能影响性能。
- 缩略图尺寸较小，很难充分表达文字动效的节奏、入场、淡出和背景适配。

适合用途：

- 当前章节的快速确认。
- 少量章节项目。
- 用户已经大概知道自己想要什么，只需要局部微调。

### 方案 B：单独做一个“文字动效实验室”页面

提供一个专门页面，集中展示全部文字风格、全部动效，并允许用户自由组合。用户可以输入示例标题、选择背景、播放预览，确认组合后回到蓝图页使用。

优点：

- 更适合创作者做风格探索和 DIY 组合。
- 预览区域可以做得更大，更接近最终视频观感。
- 可以展示全部风格、全部动效、推荐组合和不推荐组合。
- 页面职责清晰，不会把蓝图审核页挤得很复杂。

缺点：

- 用户需要跳转到另一个页面或弹层。
- 如果只做这个页面，章节卡片内仍然缺少即时反馈。
- 回到章节选择时，用户可能需要记住刚才喜欢的组合名称。

适合用途：

- 第一次使用动效系统。
- 想对比全部效果。
- 想自己组合出固定风格模板。

## 推荐最终方案：章节小预览 + 动效实验室

我建议采用混合方案，而不是二选一。

原因是两个场景不同：

- 章节卡片需要“快速确认当前选择是否合理”。
- 动效实验室需要“集中探索、对比和 DIY”。

最终体验应该是：

1. 在章节卡片内保留“风格 / 动效”选择器。
2. 在选择器旁边增加一个“预览”按钮或小缩略图。
3. 点击章节预览，打开一个轻量弹窗，播放当前章节标题的 3 秒动效预览。
4. 在蓝图审核页顶部增加“文字动效实验室”入口。
5. 实验室里展示全部风格、全部动效、推荐组合，并支持自定义组合。
6. 用户在实验室里选中组合后，可以“应用到当前章节”“应用到同类章节”“应用到全部章节”。

这个方案既不让蓝图页变重，又能让创作者有真正的试错空间。

## 蓝图审核页设计

### 章节卡片内增加内容

每个章节卡片的文字设置区建议变成：

```text
风格 [旅行明信片 v]  动效 [淡入上移 v]  [预览]  [打开实验室]
```

交互规则：

- 修改风格或动效后，章节的 `title_style` 立即更新。
- “预览”按钮打开当前章节的动态预览弹窗。
- 预览弹窗使用当前章节标题、当前风格、当前动效、当前背景模式。
- 如果没有真实背景素材，可以使用标准测试背景。
- 预览只生成低分辨率短片段或前端模拟动画，不触发完整渲染。

### 预览弹窗

弹窗内容：

- 大预览区域，比例跟当前项目画幅一致。
- 标题文字使用当前章节标题。
- 显示当前组合名称，例如“旅行明信片 + 淡入上移”。
- 提供播放 / 重播按钮。
- 提供“确认使用”和“继续调整”按钮。

建议尺寸：

- 桌面端预览宽度 720px。
- 预览时长 3 秒。
- 默认循环播放，但不要自动播放声音。

## 文字动效实验室设计

### 页面入口

入口放在蓝图审核页顶部工具区：

```text
[文字动效实验室]
```

也可以在每个章节卡片的“预览”旁边提供小入口。

### 页面结构

实验室页面建议分成三块：

```text
左侧：风格列表
中间：大预览舞台
右侧：动效列表与应用范围
```

左侧风格列表：

- 大片粗体
- 旅行明信片
- 活力弹跳
- 冲击闪切
- 极简杂志
- 自然纪录
- 浪漫柔光
- 科技未来
- 电源质感

右侧动效列表：

- 淡入上移
- 柔和推近
- 弹跳出现
- 快速冲击
- 慢速淡入推近
- 纯淡入淡出

中间预览舞台：

- 可输入测试标题。
- 可切换横屏 / 竖屏。
- 可切换测试背景：深色风景、浅色旅行、人物、城市、纯色。
- 点击任意风格或动效后立即重播。

应用按钮：

- 应用到当前章节。
- 应用到同类章节。
- 应用到全部章节。
- 保存为我的默认组合。

## 预览实现方式

### 第一阶段：前端模拟预览

用 React/CSS 在前端模拟文字层效果。

优点：

- 速度快，几乎即时。
- 不需要调用 Python、MoviePy 或 FFmpeg。
- 适合作为蓝图审核页里的快速预览。

缺点：

- 与最终 MoviePy 渲染可能有细微差异。
- 复杂字体、抗锯齿、真实背景模糊效果可能不完全一致。

适合第一版，因为目标是帮助用户快速判断方向。

### 第二阶段：真实引擎预览

增加一个 Python preview 命令，只渲染 3 秒低分辨率片段。

优点：

- 与最终成片最一致。
- 可以准确预览背景、字体、透明度、缩放、淡入淡出。

缺点：

- 需要等待几秒。
- 实现成本更高。
- 需要缓存，否则频繁点击会变慢。

建议作为第二阶段，不要一开始就做重。

## 推荐实现路线

### P1：章节卡片内预览弹窗

目标：

- 在每个章节卡片的风格 / 动效选择器旁边增加“预览”按钮。
- 点击后打开弹窗，用 CSS 模拟当前组合效果。
- 用户不用离开蓝图页即可快速确认。

验收标准：

- 每个章节都能点击预览。
- 修改风格或动效后，预览内容同步变化。
- 横屏和竖屏布局都不挤压。
- `npm run build` 通过。

### P2：文字动效实验室

目标：

- 新增一个集中浏览页面或全屏弹层。
- 支持自由组合风格和动效。
- 支持应用到当前章节、同类章节、全部章节。

验收标准：

- 所有风格和动效都能预览。
- 用户可以输入自定义测试标题。
- 用户可以把组合应用回蓝图数据。
- 不影响已有蓝图编辑流程。

### P3：真实引擎低清预览

目标：

- 增加 Python/Tauri 预览命令。
- 生成 3 秒低清 MP4 或 GIF/WebM 预览。
- 使用缓存避免重复生成。

验收标准：

- 同一风格、动效、标题、画幅、背景组合命中缓存。
- 单次预览生成时间可接受。
- 预览与最终渲染视觉一致性明显提升。

## 数据结构建议

继续使用现有字段：

```json
{
  "title_style": {
    "preset": "travel_postcard",
    "motion": "fade_slide_up",
    "color_theme": "auto",
    "position": "center"
  }
}
```

可以新增用户偏好：

```json
{
  "metadata": {
    "default_title_style": {
      "preset": "travel_postcard",
      "motion": "soft_zoom_in"
    }
  }
}
```

这样用户在实验室里保存默认组合后，后续新章节可以自动使用。

## 最终建议

先做 P1，再做 P2，最后做 P3。

最小可用版本不要直接上完整实验室。原因是当前用户最痛的点发生在蓝图审核页：看到某个章节，却不知道当前选择最终长什么样。先给每个章节一个即时预览，就能马上降低试错成本。

等 P1 稳定后，再做“文字动效实验室”。实验室适合做成创作工具，但它不是替代章节预览，而是增强探索能力。

最终产品形态应该是：

```text
章节卡片 = 快速确认
文字动效实验室 = 深度探索和 DIY
真实引擎预览 = 高保真最终确认
```

这是我认为最适合 Video Create Studio V5 的路线。
