# Video Create Studio V5 创作者拼接策略升级方案

## 目标

当前界面里的拼接方式主要是：

- FFmpeg 快速拼接
- MoviePy 交叉淡化

这两个选项更像工程实现方式，而不是视频创作者真正关心的剪辑语言。创作者关心的是成片节奏、观感、效率、稳定性，以及不同长度视频下能不能自动选到合适方案。

本方案的目标是把“拼接方式”升级为“剪辑策略”：

- 面向创作者，而不是面向底层技术。
- 适配短视频、中长视频、长视频。
- 同时提高效率、稳定性和视觉丰富度。
- 保留快速拼接的性能优势，也提供更有视频感的转场和节奏控制。
- 让用户可以先选风格，再由系统自动决定底层渲染方式。

## 当前问题

### 1. 选项太技术化

`FFmpeg 快速拼接` 和 `MoviePy 交叉淡化` 对创作者来说不够直观。

用户真正想知道的是：

- 这个模式适合快节奏还是慢节奏？
- 成片会不会高级？
- 会不会花很久？
- 长视频会不会崩？
- 素材很多时是不是合适？

### 2. 表达力不够

只有硬拼接和交叉淡化，容易出现两个极端：

- 太快：像素材堆叠，缺少剪辑感。
- 太软：全片交叉淡化，节奏容易拖、容易腻。

### 3. 不同长度视频需要不同策略

短视频更需要节奏、冲击、卡点和转场变化。

中长视频更需要章节感、呼吸感和稳定叙事。

长视频更需要性能、缓存、分段渲染、失败恢复，视觉效果要克制。

## 核心设计：从“拼接方式”升级为“剪辑策略”

建议界面不再把底层引擎作为主选项，而是提供创作者可理解的模式。

底层引擎仍然存在，但放到高级设置里。

### 新的主选项

建议把 UI 改成：

```text
剪辑策略
[智能导演] [快速成片] [旅拍柔和] [节奏卡点] [纪录叙事] [长片稳定]
```

每个策略对应一套完整决策：

- 素材时长分配
- 片段顺序节奏
- 转场类型
- 转场时长
- 章节卡出现频率
- 文字动效风格
- 音频处理策略
- 底层渲染引擎
- 长视频分段策略

## 六种推荐剪辑策略

### 1. 智能导演

默认推荐。

系统根据素材数量、视频总时长、图片/视频比例、项目画幅、章节数量自动选择策略。

适合：

- 大多数用户。
- 不确定该选什么时。
- 从短视频到长视频都能给出合理方案。

效果：

- 短视频偏节奏化。
- 中长视频偏旅拍叙事。
- 长视频自动进入稳定分段模式。

底层逻辑：

- 总时长小于 90 秒：走节奏卡点倾向。
- 90 秒到 8 分钟：走旅拍柔和或纪录叙事。
- 8 分钟以上：走长片稳定。
- 片段数量超过 80：启用分段渲染和缓存。

### 2. 快速成片

以效率为第一目标。

适合：

- 快速出样片。
- 素材预览。
- 初稿审核。
- 几千张图的大素材库快速看结果。

效果：

- 转场少。
- 片段切换直接。
- 渲染速度快。
- 失败概率低。

底层逻辑：

- 优先 FFmpeg concat。
- 尽量复用缓存。
- 少用或不用复杂 MoviePy 动画。
- 章节卡保留，但动效克制。

### 3. 旅拍柔和

适合旅行、风景、美食、生活记录。

效果：

- 轻微交叉淡化。
- 图片使用慢推近。
- 视频之间少量柔和过渡。
- 章节卡使用旅行明信片、自然纪录、大片粗体等风格。

底层逻辑：

- 转场以 soft_crossfade、fade_through_dark、gentle_push 为主。
- 转场时长 0.35 到 0.8 秒。
- 图片片段 2.5 到 4 秒。
- 视频片段根据素材长度裁剪到 3 到 7 秒。

### 4. 节奏卡点

适合短视频、运动、宠物、城市快剪、B 站/抖音节奏型内容。

效果：

- 切换更快。
- 有冲击感。
- 少用长交叉淡化。
- 多用短促转场、闪白、快速推拉、节奏切。

底层逻辑：

- 转场以 cut、quick_zoom、flash_cut、whip 为主。
- 转场时长 0.08 到 0.25 秒。
- 图片片段 0.8 到 1.8 秒。
- 视频片段 1.5 到 3.5 秒。
- 未来可接 BGM 节拍检测。

### 5. 纪录叙事

适合中长视频、城市游记、家庭记录、展览、博物馆、自然观察。

效果：

- 节奏稳定。
- 转场克制。
- 章节结构清晰。
- 重视标题、日期、地点、景点之间的层级。

底层逻辑：

- 转场以 cut、fade、soft_crossfade 为主。
- 转场时长 0.25 到 0.5 秒。
- 章节卡停留更久。
- 同章节素材之间少转场，跨章节才明显转场。
- 避免过度动效打断叙事。

### 6. 长片稳定

适合 8 分钟以上、80 个以上片段、几千张图片或几十段视频的大项目。

效果：

- 视觉克制。
- 渲染稳定。
- 支持断点恢复。
- 不追求每个片段都有复杂转场。

底层逻辑：

- 强制启用 V5.6 分段渲染。
- 每 3 到 5 分钟一个 chunk。
- chunk 内可用轻量转场。
- chunk 之间使用 FFmpeg concat copy。
- 失败后只重渲染失败 chunk。

## 推荐的转场类型库

### 基础转场

`cut`

直接切。速度快，适合大多数场景。

`crossfade`

交叉淡化。适合旅行、风景、纪实，但不能滥用。

`fade_through_dark`

淡出到暗再淡入。适合章节切换、地点变化、时间变化。

`fade_through_white`

闪白。适合高光、运动、阳光、快节奏片段。

### 运动转场

`gentle_push`

轻微推近或推远。适合图片和旅行素材。

`quick_zoom`

快速推拉。适合节奏卡点、运动、城市。

`whip_pan`

横向甩动。适合同类素材之间的快速切换。

### 章节转场

`chapter_card`

插入章节卡。

`title_overlay`

在素材上叠加地点/景点标题，不打断视频。

`bridge_blur`

用上一段和下一段素材做模糊桥接。适合旅行章节过渡。

## 不同长度视频的推荐配置

### 短视频：15 秒到 90 秒

推荐策略：

- 智能导演
- 节奏卡点
- 旅拍柔和

推荐参数：

```json
{
  "target_duration": "15-90s",
  "image_duration": "0.8-2.0s",
  "video_clip_duration": "1.5-4.0s",
  "transition_duration": "0.08-0.35s",
  "chapter_card_duration": "1.2-2.0s",
  "render_mode": "standard"
}
```

核心原则：

- 少废话，快进入高光。
- 每 5 到 12 秒需要一个节奏变化。
- 不建议大量完整章节卡，更多使用标题叠加。

### 中长视频：90 秒到 8 分钟

推荐策略：

- 旅拍柔和
- 纪录叙事
- 智能导演

推荐参数：

```json
{
  "target_duration": "90s-8min",
  "image_duration": "2.0-4.0s",
  "video_clip_duration": "3.0-8.0s",
  "transition_duration": "0.25-0.65s",
  "chapter_card_duration": "2.0-3.0s",
  "render_mode": "standard_or_auto_stable"
}
```

核心原则：

- 章节结构要清楚。
- 转场要有变化，但不要抢内容。
- 景点标题可以用 overlay，城市/日期用完整章节卡。

### 长视频：8 分钟以上

推荐策略：

- 长片稳定
- 纪录叙事
- 智能导演

推荐参数：

```json
{
  "target_duration": "8min+",
  "image_duration": "2.5-5.0s",
  "video_clip_duration": "4.0-12.0s",
  "transition_duration": "0.15-0.45s",
  "chapter_card_duration": "2.5-3.5s",
  "render_mode": "long_stable",
  "chunk_seconds": 240
}
```

核心原则：

- 不要让每个片段都有复杂转场。
- 稳定性优先于花哨。
- 分段缓存必须打开。
- 失败后能继续，不要整片重来。

## 智能导演决策树

建议系统自动计算以下指标：

```json
{
  "asset_count": 420,
  "image_count": 360,
  "video_count": 60,
  "estimated_duration": 780,
  "section_count": 18,
  "avg_assets_per_section": 23,
  "portrait_ratio": 0.35,
  "video_ratio": 0.14
}
```

然后自动选择：

```text
if estimated_duration >= 480s or asset_count >= 300:
  use 长片稳定
elif estimated_duration <= 90s and asset_count >= 20:
  use 节奏卡点
elif video_ratio >= 0.45:
  use 纪录叙事
elif section_count >= 5:
  use 旅拍柔和
else:
  use 快速成片
```

## 数据结构建议

### Render Plan 新增字段

建议在 `render_settings` 中加入：

```json
{
  "edit_strategy": "smart_director",
  "transition_profile": "travel_soft",
  "rhythm_profile": "medium",
  "target_duration_policy": "auto",
  "render_mode": "auto",
  "chunk_seconds": 240
}
```

### Segment 新增字段

建议每个片段支持：

```json
{
  "transition": {
    "type": "soft_crossfade",
    "duration": 0.45,
    "scope": "in"
  },
  "motion": {
    "type": "ken_burns",
    "intensity": "low"
  },
  "rhythm": {
    "role": "establishing",
    "importance": 0.82
  }
}
```

### Blueprint Metadata 新增字段

建议用户在蓝图阶段就能选择：

```json
{
  "metadata": {
    "edit_strategy": "travel_soft",
    "transition_profile": "balanced",
    "duration_profile": "medium",
    "render_mode": "auto"
  }
}
```

## UI 建议

### 当前 UI 改造

把现在的：

```text
渲染引擎
[自动选择 / FFmpeg 快速拼接 / MoviePy 交叉淡化]
```

升级为：

```text
剪辑策略
[智能导演 / 快速成片 / 旅拍柔和 / 节奏卡点 / 纪录叙事 / 长片稳定]
```

下面增加策略说明：

```text
智能导演：根据素材规模和目标时长自动选择节奏、转场与稳定渲染模式。
```

高级设置里再放：

```text
底层渲染方式
[自动 / 标准 MoviePy / 长片稳定 / FFmpeg 快速拼接]
```

### 蓝图页增强

每个章节可以显示：

```text
章节节奏：[舒缓 / 标准 / 快节奏]
章节转场：[跟随全局 / 柔和 / 快切 / 章节卡]
```

这样用户可以全局选一个策略，再对个别章节做微调。

## 实现路线

### P1：策略命名和 UI 改造

目标：

- 把技术型选项改成创作者型“剪辑策略”。
- 保留旧引擎作为高级设置。
- 先不大改渲染核心。

验收：

- 用户能选择六种剪辑策略。
- 参数写入 blueprint metadata。
- 编译 render_plan 时能看到 `edit_strategy`。

### P2：Render Plan 加转场配置

目标：

- `compile` 阶段根据策略给每个 segment 写入 transition/motion/rhythm。
- 图片片段自动加轻微运动。
- 跨章节使用不同转场。

验收：

- Render Plan 中每个片段有清晰转场配置。
- 不同策略生成的配置明显不同。
- smoke test 覆盖短视频、中长视频、长视频。

### P3：实现轻量转场库

目标：

- 先实现稳定、低风险的转场：
  `cut`、`crossfade`、`fade_through_dark`、`fade_through_white`、`gentle_push`、`quick_zoom`。

验收：

- 标准 MoviePy 模式可用。
- 长片稳定模式可降级使用轻量转场。
- 转场时长自动限制，不超过片段时长 25%。

### P4：长片稳定策略升级

目标：

- 长片使用 chunk 内轻量转场。
- chunk 间使用 FFmpeg concat。
- 失败后只重渲染失败 chunk。

验收：

- 10 分钟以上项目可稳定完成。
- 中途失败可以复用已完成 chunk。
- 构建报告说明使用了哪些缓存。

### P5：创作者预览

目标：

- 在蓝图阶段预览某个章节的拼接节奏。
- 类似文字动效实验室，做“剪辑策略预览”。

验收：

- 选择策略后能生成 10 到 20 秒低清片段。
- 用户可在完整渲染前判断节奏是否合适。

## 推荐优先级

最建议先做：

1. UI 从“渲染引擎”改为“剪辑策略”。
2. 新增 `edit_strategy` 数据结构。
3. 让 compile 阶段生成 transition/rhythm 配置。
4. 先实现 `cut`、`crossfade`、`fade_through_dark`、`gentle_push`。
5. 再扩展节奏卡点和真实预览。

## 最终建议

不要把“FFmpeg 快速拼接”和“MoviePy 交叉淡化”继续作为主体验。

它们应该变成底层实现细节。

主体验应该是创作者语言：

```text
我想快速出片
我想旅拍柔和
我想节奏卡点
我想纪录叙事
我想长片稳定
我交给智能导演
```

这样用户会更容易做选择，也更容易获得灵感。系统负责把这些选择翻译成渲染策略、转场配置、缓存策略和性能策略。

这是更适合 Video Create Studio V5 的长期方向。
