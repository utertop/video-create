# Video Create Studio V5 音频稳定混音策略说明

## 1. 目的

这份文档解决一个很具体的问题：

在长视频、海量素材、稳定优先渲染场景下，音频系统需要更稳定、更可预期，但又不能因为“性能优化”把视频的氛围、质感、节奏感做没。

核心原则只有一句话：

**允许降级执行方式，不允许粗暴降级创作表达。**

也就是说：

- 可以降级的是混音实现复杂度
- 不能轻易降级的是音乐存在感、原声保留、情绪节奏、整体氛围

## 2. 设计结论

音频系统后续不应该叫“音频降级策略”，更适合叫：

**音频稳定混音模式**

原因很简单：

- “降级”这个词容易让创作者理解成“音质变差、气氛变弱、音乐白加了”
- 实际想做的是“同样保留创作意图，但换更稳的执行路径”

所以对外表达建议统一为三档：

- `稳定优先`
- `平衡推荐`
- `质感优先`

## 3. 哪些不能轻易降

下面这些属于创作意图层，默认不能因为性能理由被直接拿掉：

- 是否有 BGM
- 是否保留视频原声
- BGM 的整体存在感
- 章节之间的情绪连续性
- 基础淡入淡出
- 原声与 BGM 的基本层次关系

如果这些被简单砍掉，用户会明显感觉：

- 视频“能播但没感觉”
- 音乐加了等于没加
- 氛围变平
- 节奏感不足

## 4. 哪些可以降

下面这些属于执行层，可以根据机器配置、时长、素材量做稳定化处理：

- 混音路径选择
- 实时动态处理强度
- ducking 的精细程度
- 音量包络点数量
- 长视频里的连续滤镜复杂度
- 最终混音时是否优先复用缓存和 FFmpeg

这类调整的目标不是“削弱质感”，而是：

- 提高成功率
- 降低内存峰值
- 控制最终收尾耗时
- 减少最后一步失败

## 5. 三档策略

### 5.1 稳定优先

适用场景：

- 30 分钟以上长视频
- 数千图片或大量视频素材
- 中低配置机器
- 用户更看重“先稳定出片”

保留内容：

- 保留 BGM
- 保留原声
- 保留基础淡入淡出
- 保留基础 ducking
- 保留整体情绪层次

执行策略：

- 优先走 FFmpeg 流式混音
- 优先复用 `audio_cache`
- ducking 使用保守固定值，不做复杂动态自动化
- 淡入淡出只保留简单首尾包络
- 避免复杂多段音量曲线
- 避免最后一步再进入大规模 MoviePy 音频时间线

限制原则：

- 不能把 BGM 直接压到“几乎听不见”
- 不能默认关掉原声
- 不能把所有内容都混成一层平音

### 5.2 平衡推荐

适用场景：

- 大多数日常项目
- 中短视频和中长视频
- 用户希望稳定与质感都兼顾

保留内容：

- 完整 BGM
- 完整原声
- 正常淡入淡出
- 正常 ducking
- 基本的段落层次

执行策略：

- 能走 FFmpeg 就走 FFmpeg
- 必要时允许适度滤镜处理
- 保留大多数混音效果
- 控制滤镜复杂度，不堆过多自动化

这是默认推荐档位。

### 5.3 质感优先

适用场景：

- 短视频
- 预览确认后的小批量正式输出
- 高配置机器
- 用户明确追求细节表现

保留内容：

- 最完整的音乐表现
- 更自然的原声与 BGM 层次
- 更细的淡入淡出和情绪衔接
- 更灵活的 ducking 表达

执行策略：

- 允许更复杂的混音滤镜
- 允许更丰富的包络处理
- 允许较高的最终收尾成本

风险提示：

- 长视频下耗时更高
- 最终混音阶段可能更慢
- 对机器内存和 CPU 更敏感

## 6. 推荐的真实降级边界

后续实现时，建议遵守下面这条边界线。

### 允许做的“稳定化”

- 把实时 ducking 简化成固定 ducking
- 把复杂音量包络简化成少量关键点
- 把多层处理改成单层 FFmpeg filter
- 把源音轨与 BGM 统一缓存成标准格式后再混
- 把最终混音优先放在 FFmpeg 流式阶段完成

### 不允许默认做的“伤质感”

- 默认移除 BGM
- 默认关闭原声
- 把 BGM 音量压得接近不可感知
- 去掉所有淡入淡出
- 把不同章节都混成一个死板的平层

## 7. UI 表达建议

UI 不建议写成：

- `音频降级`
- `音频简化`

更合适的表达：

- `音频混音模式`
- `音频稳定模式`
- `当前混音策略`

推荐说明文案：

- `稳定优先：保留音乐和原声层次，使用更稳的混音路径，适合长视频与大批量素材。`
- `平衡推荐：兼顾质感与效率，适合大多数项目。`
- `质感优先：保留更完整的混音细节，适合短片和高质量输出。`

## 8. 风险提示文案

当检测到长视频或大量素材时，建议提示：

`当前素材量较大，系统将优先使用稳定混音路径。音乐与原声仍会保留，但部分复杂动态处理会自动简化，以降低生成失败风险。`

如果用户手动切到 `质感优先`，建议提示：

`当前项目较大，质感优先模式可能显著增加最终混音耗时，并提高失败风险。`

## 9. 实现优先级建议

### P1

- 先统一音频缓存
- 先统一 FFmpeg 优先混音路径
- 先把 BGM、原声、chunk 音轨尽量留在 FFmpeg 体系

### P2

- 再做三档混音模式参数化
- 把 `稳定优先 / 平衡推荐 / 质感优先` 显式暴露到 UI
- 根据视频时长、素材量、性能档位做默认推荐

### P3

- 最后再补更细的动态包络策略
- 做预览和正式渲染的一致性校验
- 做更多创作者可理解的风险提示

## 10. 最终结论

会存在“音频优化过头导致质感下降”的风险，但这不是因为做了稳定策略本身，而是因为策略边界没画清楚。

正确做法不是：

- 为了稳定，把音乐做弱

正确做法应该是：

- 为了稳定，保留音乐表达，简化混音执行

所以后续音频性能优化必须坚持这条原则：

**稳定模式优化的是技术路径，不是牺牲创作表达。**
